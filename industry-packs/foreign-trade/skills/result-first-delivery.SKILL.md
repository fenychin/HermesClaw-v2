# 结果优先交付协议技能 (result-first-delivery)

## 1. 技能概述
本技能是 HermesClaw 外贸行业包中的核心高复用协议技能。它主要作为一种**系统交付规约**被任意数字员工（Agent）继承或绑定，强制智能体在进行信息回复、邮件起草、报价单撰写或决策汇报时，遵循**「先给结果，后给理由」**的黄金交付规约，保障在高度流式传输或终端用户查阅时能够一目了然，杜绝冗长拖沓的推理前置。

## 2. 核心特征
- **先结果后理由**：强制性约束首段输出核心交付件（结论、回复草稿或计算数值），然后才在下方展开陈述分析、推导与事实依据。
- **双语代码块注入**：自动注入规范化的邮件交付格式，强制生成分别包裹在 ````email-cn```` 与 ````email-en```` 专属语法块中的双语邮件。
- **知识库优先级规则**：内建知识库优先集成策略，在发生事实或策略冲突时，优先采用并标注企业私有知识库的内容，提供清晰的引用文献。
- **自进化评估反馈**：当技能在多次调用中出现用户负向纠偏时，系统将自动产生提案建议并触发相应的升级门禁。

## 3. 系统内置规则
当任何智能体（Agent）绑定或触发此技能时，系统将自动前置注入以下系统指令（System Rules）：
1. **结果优先**：在首行无废话直接给出用户所求的核心内容（如已生成的开发信、报价单等），严禁将冗长的原因解释、环境介绍置于内容之前。
2. **代码块隔离**：所有物理写出的邮件书信，必须使用 markdown 代码块进行显式隔离，分别注明中文与英文，如：
   ```email-cn
   // 中文信件内容
   ```
   ```email-en
   // 英文信件内容
   ```
3. **溯源引用**：涉及数据及策略时，需指明是否基于本地知识库或属于通用模型推理。
4. **Harness 进化日志**：在响应最末尾输出简短的系统进化信息，便于平台审计。
