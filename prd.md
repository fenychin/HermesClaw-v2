# 1. 产品概述

## 1.1 产品名称
HermesClaw-v2

## 1.2 产品定位
HermesClaw-v2 是一个面向中小企业（小B场景）的 AI 数字员工基础平台，以 Web 工作台作为主交互入口，以 Hermes 作为云端规划与记忆控制面，以 OpenClaw 作为用户移动端执行与连接器数据面，帮助企业快速构建、管理、升级可持续工作的 AI 数字员工系统。

平台不仅提供通用 AI 对话能力，更提供：
- 行业工作流生成
- 多智能体协同
- 企业知识记忆沉淀
- 文件与连接器统一接入
- 动态 Harness 自我进化与升级
- 项目空间化管理

首个优先切入行业为：**外贸**。

---

# 2. 产品愿景

构建一个适用于中小企业的 AI-first 数字员工操作系统，让企业可以像管理团队一样管理智能体、知识、连接器、项目与工作流，并通过动态 Harness 工程持续自我优化、自我进化。

---

# 3. 核心理念

## 3.1 AI-first
系统以 AI 为第一执行主体，人类负责目标设定、边界治理、审批与复盘。

## 3.2 双层架构
- Hermes：云端控制面，负责规划、记忆、编排、策略与进化
- OpenClaw：移动端数据面，负责执行、采集、连接器动作、现场反馈

## 3.3 动态 Harness
平台不采用静态写死的 Agent 工作流，而是支持：
- 根据行业自动生成 Workflow
- 根据运行数据自动评估瓶颈
- 提出升级提案
- 审批后动态升级 Harness

## 3.4 行业优先
不是做一个通用聊天工具，而是做一个可快速进入垂直行业的数字员工平台。外贸为第一优先行业模板。

# 5. 核心问题

当前中小企业在 AI 使用上存在以下问题：
1. AI 只能聊天，无法形成工作闭环
2. 知识、文件、任务、客户信息分散，无法沉淀组织记忆
3. 多个工具孤立，连接器能力弱
4. 没有行业工作流与岗位化 Agent
5. 无法根据业务运行结果持续升级 AI 系统
6. 缺少适合企业真实业务场景的统一工作台

HermesClaw-v2 要解决的核心问题是：
**让 AI 从一次性辅助工具，升级为可管理、可执行、可记忆、可进化的企业数字员工系统。**

# 8. 核心业务闭环

1. 用户在 Web 工作台发起任务
2. Hermes 识别意图，调用行业模板与企业知识
3. 系统生成任务 Workflow 和当前动态 Harness
4. 智能体执行任务
5. OpenClaw 调用连接器或移动端能力完成动作
6. 执行结果与日志回传 Hermes
7. Hermes 更新短中长期记忆
8. 若检测到失败、低效或知识缺口，触发 Harness 评估
9. 生成升级提案并进入审批
10. 审批通过后动态更新 Workflow / Harness

# 11. 动态 Harness 引擎

## 11.1 定义
动态 Harness 是平台的核心工程机制，用于把 Agent 从静态配置升级为可演化系统。

## 11.2 组成
- 任务边界
- 上下文供给
- 工具接入
- 反馈回路
- 安全护栏
- 进化调度

## 11.3 功能目标
- 为不同企业、不同项目、不同岗位、不同场景生成适配的工作流
- 根据执行结果自动发现瓶颈
- 生成升级提案
- 经审批后动态升级

## 11.4 升级触发条件
- 同类任务失败率异常
- 工具调用成功率下降
- 知识缺口明显
- 用户多次手动修正
- 新连接器接入完成
- 新行业模板启用

## 11.5 升级流程
1. 采集执行数据
2. 生成评估报告
3. 输出升级提案
4. 人工审批
5. 生效与回滚监控

# 15. 风险与约束

## 15.1 风险
- 外贸行业数据源复杂
- 企业连接器授权门槛较高
- 智能体效果受知识质量影响
- 动态 Harness 需要严格审批与回滚控制

## 15.2 约束
- 必须遵循 AGENTS.md 作为最高规则
- 不允许模型直接绕过 Harness
- 所有高风险动作必须审批
- 所有执行必须可回溯、可审计
